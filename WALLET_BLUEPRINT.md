# Aura Wallet: Technical Blueprint

## 1. โครงสร้างกุญแจและการเข้ารหัส (Key & Encryption)
- **Asymmetric Cryptography:** ใช้ระบบกุญแจคู่ (Private Key สำหรับอนุมัติการโอน และ Public Key สำหรับสร้าง Address)
- **Standard BIP-39:** ใช้ชุดคำศัพท์ 12-24 คำ (Mnemonic Seed) เป็นต้นกำเนิดกุญแจ เพื่อความมาตรฐานในการสำรองและกู้คืนข้อมูล
- **SHA-256 Hashing:** ใช้การเข้ารหัสระดับเดียวกับกองทัพ เพื่อแปลง Public Key ให้เป็น **AUR Address** (ป้องกันการย้อนกลับไปหาตัวกุญแจหลัก)

## 2. มาตรฐานความปลอดภัยของธุรกรรม (Transaction Security)
- **Self-Custody (กุญแจอยู่ที่ผู้ใช้):** ระบบถูกออกแบบให้ Private Key ถูกเข้ารหัสและเก็บไว้ในฝั่งผู้ใช้ (Local Storage) เท่านั้น **ไม่มีการส่งกุญแจเข้าสู่ระบบส่วนกลาง**
- **Multi-Signature (Multi-Sig):** รองรับระบบที่ต้องใช้มากกว่า 1 กุญแจในการอนุมัติโอน (เช่น 2 ใน 3 กุญแจ) เพื่อป้องกันกรณีอุปกรณ์ใดอุปกรณ์หนึ่งสูญหายหรือถูกแฮ็ก
- **Digital Signature:** ทุกธุรกรรมต้องมีการ "ลงลายเซ็นดิจิทัล" ก่อนส่งเข้าสู่เน็ตเวิร์ก เพื่อยืนยันว่าเป็นเจ้าของเหรียญตัวจริง

## 3. การจัดการหน่วยย่อย (Unit Architecture)
- **18 Decimal Places:** ระบบรองรับทศนิยมสูงสุด 18 ตำแหน่ง เพื่อความละเอียดในการโอนระดับเล็กมาก
- **Atomic Unit (อะตอม):** กำหนดให้หน่วยที่เล็กที่สุดเรียกว่า "อะตอม" (1 AUR = 10^18 อะตอม) เพื่อความแม่นยำทางคณิตศาสตร์ในระบบ Blockchain

## 4. ระบบการโอนและค่าธรรมเนียม (Transfer & Fee)
- **1% Burn/Fee:** ทุกการโอนจะถูกหัก 1% โดยอัตโนมัติจากยอดโอนตามกฎของ Smart Contract
- **Minimum Transfer:** ระบบรองรับการโอนแบบไม่มีขั้นต่ำ (ตราบใดที่ครอบคลุมค่าธรรมเนียม)

---
*Blueprint สรุปแนวทางสำหรับระบบจัดเก็บที่ผู้ใช้ถือ "กุญแจ" เอง 100% (Non-Custodial) ใช้มาตรฐานการเข้ารหัสสากล (SHA-256) และบริหารจัดการหน่วยย่อยแบบละเอียด (18 ทศนิยม) เพื่อความปลอดภัยและความโปร่งใสสูงสุด*
